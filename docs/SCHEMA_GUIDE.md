# PropertyModel JSON Guide (`.icprop.json`)

A human-readable, field-by-field guide to the property input document — the file the
application loads as one of its **only two intake surfaces** (spec §5.4; the other is the
rent roll Excel template). It is written for people and external extraction workflows that
produce these files *outside* the application; the app itself knows nothing about how a
document was created.

**Standing principle:** any JSON produced by an external extraction workflow is reviewed by
a human against the source document before it is used for calculation.

The machine-checkable companion is [`property_model.schema.json`](property_model.schema.json)
(formal JSON Schema, generated by `scripts/export_json_schema.py`). **Both files are
regenerated whenever the pydantic models in `engine/models/` change**;
`tests/unit/test_schema_docs.py` fails when they drift.

## Validating a document

```python
from engine.models import load_property
model = load_property("my_deal.icprop.json")   # raises pydantic.ValidationError on problems
```

Validation errors name the exact field path and the problem, e.g.
`rent_roll.0.end_date: end_date must be after start_date`. Unknown/misspelled keys are
**rejected**, not ignored — a typo cannot silently drop an input.

## Conventions (read first)

- **Dates** are ISO strings, `"YYYY-MM-DD"`. `analysis_begin` must be the first day of a
  month; all other timing snaps to months.
- **Rates and percentages are entered as percents**: `3.0` means 3%, not 0.03. This applies
  to inflation rates, vacancy rates, renewal probability (0–100), `pct_fixed` (0–100),
  discount and cap rates, loan rates, admin fees, and every other `pct` field.
- **Money** is USD; **areas** are square feet (SF).
- **Refs** are plain strings naming another object in the same document (a market leasing
  profile's `name`, an expense's `name`, …). Every ref is checked at load time; an unknown
  name is a validation error. Names within a collection must be unique.
- **Annual rate schedules** (`[{"year": 1, "rate": 3.0}, …]`) carry the last entered rate
  forward; years before the first entry contribute 0%. `year` is a 1-based analysis year, or
  a calendar year (e.g. `2027`) where the owning model's timing basis says so.
- **Omit what you don't know.** Optional fields have sensible defaults (listed below);
  `null` and omission are equivalent. Required fields are marked **req**.

### Shared building blocks

**`YearRate`** — one year in an annual schedule: `{"year": 1, "rate": 3.0}`.

**`MoneyRate`** — an amount with a unit: `{"amount": 5.75, "unit": "dollars_per_area_per_year"}`.

**`MoneyUnit`** values:

| Value | Meaning |
|---|---|
| `dollars_per_area_per_year` | $/SF/yr |
| `dollars_per_area_per_month` | $/SF/mo |
| `dollars_per_year` | $/yr |
| `dollars_per_month` | $/mo |
| `dollars_per_area` | $/SF one-time (TIs) |
| `dollars` | $ one-time |
| `pct_of_market` | % of market rent (needs a market leasing profile) |
| `pct_of_last_rent` | % of the prior contract rent |

**`Timing`** — when a revenue/expense item applies:

| Field | Type / values | Default | Notes |
|---|---|---|---|
| `method` | `continuous` \| `date_range` \| `repeating` | `continuous` | |
| `start`, `end` | date | null | `date_range` requires `start` |
| `repeat_months` | list of 1–12 | null | calendar months in which the amount posts |
| `repeat_every_months` | int ≥ 1 | null | post every N months from `start` |

`repeating` requires `repeat_months` or `repeat_every_months`.

**`Limits`** — per-period clamps: `{"min": null, "max": 30000.0}`.

**Inflation override** (any field named `inflation`, `growth`, or `fixed_inflation`) — one of:
a built-in index name (`"general"`, `"market_rent"`, `"expense"`, `"cpi"`), a custom index
name (see `inflation.custom_indices`), an inline schedule (`[{"year": 1, "rate": 2.0}]`), or
`null` for the item's default index.

**`RentStep`** — one step in a rent schedule. Give exactly one of `date` / `month_offset`
(months from segment start, ≥ 0) and exactly one of `amount` (with `unit`) / `pct_increase`:
`{"date": "2029-07-01", "pct_increase": 10.0}` or
`{"month_offset": 12, "amount": 29.0, "unit": "dollars_per_area_per_year"}`.

## Expenses (`expenses[]`, `expense_groups[]`)

Expense items use the same shape as property revenues, plus:

| Field | Type / values | Default | Notes |
|---|---|---|---|
| `category` | `operating` \| `non_operating` \| `capital` | `operating` | operating is above NOI; capital after NOI; non-operating below the line |
| `recoverable` | bool | null → category default | operating default `true`; capital/non-op `false` |
| `annual_overrides` | list of `{year, amount}` | `[]` | known per-year actuals: for a fiscal year with an override, `amount` is used directly (posted `amount/12` per active month) in place of the computed base × inflation — the override wins completely and is **not** re-clamped by `limits`; other years compute as usual. `year` is the fiscal-year label (= the calendar year for a December fiscal-year-end). Escape hatch for tax abatements / reassessments / any known figure (DEVIATIONS.md §12) |
| `expense_group` | ref | null | reporting rollup / recovery pool membership |
| `amortization_years` | int ≥ 1 | null | **capital only**: recover over N years |
| `refundable` | bool | `false` | **capital only** |

Expense `unit` values are the revenue units minus `spaces_times_rate`.
`expense_groups`: `{"name": …, "members": ["Cleaning", "Utilities"]}` (members are expense names).

"free_rent_abates": false}` — negative `amount` = abatement.

**Security deposit**: `{"amount": 1.0, "unit": "months_of_rent" | "dollars" | "dollars_per_area",
"refunded_at_expiration": true}`.

## Recovery structures (`recovery_structures[]`)

User-defined structures: `{"name", "pools": [...]}`. Caps/floors and expense
adjustments belong to **each pool** (spec §3.14), so a capped CAM pool can sit
beside an uncapped tax pool in one structure.

**Pool** fields:

| Field | Type / values | Default | Notes |
|---|---|---|---|
| `expenses` | list of refs | **req** | expense names or expense group names |
| `method` | `net` \| `stop` \| `base_year` \| `fixed` | `net` | |
| `gross_up_pct` | float (0, 100] | null | gross variable expenses up to a stipulated occupancy; never gross down |
| `base_amount_per_area` | float | null | required for `stop` |
| `base_year` | `{year, fiscal, gross_up_pct, known_amount}` | null | `year` null = analysis year 1; a stated `year` ending before the analysis start falls back to analysis year 1. `known_amount` = the known frozen base-year pool as a **total annual dollar** figure (used directly, bypassing the window; `year` then documents which year it represents) |
| `fixed_amount` | float | null | required for `fixed` |
| `fixed_inflation` | inflation override | null | |
| `admin_fee_pct` | float | `0.0` | |
| `admin_fee_applies` | `before_stop` \| `after_stop` | `before_stop` | |
| `denominator` | `rentable_area` \| `property_size` \| `occupied_area` \| `fixed_area` | `rentable_area` | |
| `denominator_fixed_area` | float > 0 | null | required for `fixed_area` |
| `pro_rata_share_override` | float | null | percent |
| `caps_floors` | see below | null | per pool |
| `expense_adjustments` | list | `[]` | per pool |

`caps_floors`: `{"yearly_cap_pct", "cumulative_cap_pct", "min", "max"}` (all optional).
`expense_adjustments`: list of `{"expense": ref, "action": "exclude" | "add", "pct": 100.0}`.

## Worked example

A complete, minimal-but-representative document: a 100,000 SF single-tenant industrial
net-lease deal with three operating expenses recovered net, a rollover profile, purchase,
one fixed loan, and DCF/resale assumptions. Sections not shown (`general_vacancy`,
`credit_loss`, absorption, % rent, recovery structures, …) simply take their defaults.
This example is validated by `tests/unit/test_schema_docs.py` on every test run.
